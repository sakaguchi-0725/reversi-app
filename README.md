# アプリケーションの仕様を決める  
## 仕様検討の進め方
- どんな機能を作り、どんな機能を作らないかを決める
- 文字で整理するだけでは、認識齟齬や考慮漏れが生まれやすいため、作図をすることが有効

以下5つの図表を作成  
- コンテキスト図: 開発するシステムと関わる人や他システムを整理した図
- ユースケース図: 登場人物がシステムで何をするか整理した図
- クラス図（概念モデル）: 機能の洗い出しや認識合わせに有効
- 機能一覧
- 画面イメージ
